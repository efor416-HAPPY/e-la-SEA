# 📌 네이버 서치어드바이저 API 연동 필수 점검 체크리스트

네이버 서치어드바이저 제휴 API 연동 시 빈번하게 발생하는 오류와 이를 미연에 방지하기 위한 점검 항목들입니다.

---

## 1. 도메인 및 URL 일치 여부 점검 (가장 중요)
* [ ] **소유 확인 완료 도메인과 스키마(Schema) 일치**
  * 서치어드바이저 웹마스터 도구에 소유 확인을 마친 대표 주소와 API 요청 URL의 `http` / `https` 프로토콜이 정확히 일치해야 합니다.
* [ ] **`www` 서브도메인 포함 여부 일치**
  * 등록된 사이트 주소가 `https://your-site.com` 이라면 요청 URL도 `https://your-site.com/post` 여야 합니다. `https://www.your-site.com/post` 로 요청하면 등록되지 않은 사이트로 간주되어 에러가 발생합니다.
* [ ] **하위 경로(Subpath) 기준 준수**
  * 요청하는 URL은 소유권이 확인된 도메인의 하위 경로 내에 존재해야 합니다.

## 2. API 인증 및 권한(Authorization) 점검
* [ ] **Access Token 복사 상태 확인**
  * 토큰 앞뒤에 공백(Space)이 붙어 복사되지 않았는지 확인합니다.
* [ ] **Authorization 헤더의 포맷 확인**
  * 헤더에 `Authorization: Bearer <Access_Token>` 형태로 들어가야 하며, `Bearer` 키워드 뒤에 **공백 1칸**이 반드시 있어야 합니다.
* [ ] **토큰 만료 여부**
  * 발급받은 액세스 토큰은 네이버 측 정책에 의해 만료되거나 변경될 수 있습니다. 만약 지속적으로 `401 Unauthorized` 또는 인증 에러가 발생한다면 서치어드바이저 도구 설정 화면에서 토큰을 재발급 받으시기 바랍니다.

## 3. 웹 서버 환경 및 네트워크 설정 점검
* [ ] **리다이렉트(Redirect) 처리 여부**
  * 수집을 요청한 URL이 다른 URL로 리다이렉트(예: `301 Moved Permanently`, `302 Found`)되는 경우, 네이버 수집기가 정상적으로 원본 문서를 수집하지 못해 오류로 처리되거나 유실될 수 있습니다. 반드시 최종 랜딩 페이지 URL로 요청하세요.
* [ ] **방화벽 및 접근 제어(IP 차단)**
  * 네이버 수집 로봇(NaverBot)의 접근이 서버 방화벽이나 웹 애플리케이션 방화벽(WAF), 또는 `robots.txt` 파일에 의해 차단되어 있지 않은지 확인하세요.
* [ ] **robots.txt 설정 검증**
  * `robots.txt` 설정이 네이버봇(`Yeti` 또는 `NaverBot`)의 수집을 허용하고 있는지 확인합니다.
    ```txt
    User-agent: Yeti
    Allow: /
    ```

## 4. API 요청 규격 점검
* [ ] **JSON 페이로드 구조**
  * 요청 Body는 JSON 형식이어야 하며, 헤더에 `Content-Type: application/json`이 명시되어야 합니다.
  * 페이로드의 `urls` 키는 배열 형태여야 하며, 각 아이템은 `url`과 `type` 필드를 포함해야 합니다.
    ```json
    {
      "urls": [
        {
          "url": "https://your-site.com/post",
          "type": "update"
        }
      ]
    }
    ```
* [ ] **요청 제한(Quota) 확인**
  * 네이버 서치어드바이저 API는 일일 수집 요청 한도가 설정되어 있을 수 있습니다. 일일 할당량을 초과한 요청은 에러를 반환하므로 주의해야 합니다.
